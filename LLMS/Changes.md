File purpose: Track feature requests, bugs, refactors, and investigations that should later be implemented or carried out here.

Guidelines:
- One item per entry with a clear title and short description.
- Include priority and impact if known (e.g., High/Med/Low).
- Mark status explicitly (e.g., `todo`, `done`).
- Keep this list up to date; remove or archive items that are resolved.
- When an item references a design in `LLMS/Design`, keep the design link current.
- Before implementing an item that has a design, review that design for unresolved review points, open questions, risks, follow-ups, or proposed defaults that still need user confirmation.
- If those unresolved points materially affect scope, behavior, data shape, UX, or rollout, ask the user to answer or approve them before implementation.
- When implementing an item from this file, update all relevant project documentation in `LLMS/Documentation` as part of the same change.
- If documentation is added, moved, or removed during implementation, update `LLMS/Documentation/AGENTS-INDEX.md` too.
